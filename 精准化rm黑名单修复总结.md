## 问题诊断

### 错误现象
切换代理模式时再次失败，出现 `[Security] 拦截到危险操作` 错误

### 根本原因
之前的黑名单正则表达式在匹配系统关键目录时存在缺陷：

```javascript
// 旧规则（过度宽泛）
/\brm\s+(-[a-z]*\s+)*\/(data|etc|root|...) /
```

此规则会拦截以下**安全操作**：
- `rm -f /data/ShellCrash/.start_error` ❌ (误拦截)
- `rm /data/tmp/*` ❌ (误拦截)

原因是正则表达式在检查 `/data` 时，只要目录名出现就匹配，不区分"删除整个目录"vs"删除目录内文件"

---

## 修复方案

### 修改内容

**文件**: `src/utils/validators.js` (第 63 行)

```javascript
// 新规则（精准针对）
/\brm\s+(-[a-z]*\s+)*(\/data\s|\/data$|\/etc\s|\/etc$|\/root\s|\/root$|\/sys\s|\/sys$|\/usr\s|\/usr$|\/lib\s|\/lib$|\/bin\s|\/bin$|\/sbin\s|\/sbin$|\/boot\s|\/boot$)/
```

### 核心改进

关键改变：每个目录后加 `\s|...$`

| 模式 | 匹配 | 说明 |
|------|------|------|
| `/data$` | `rm -rf /data` ❌ | 只拦截删除整个目录 |
| `/data\s` | `rm -rf /data /other` ❌ | 删除多个目录 |
| ~~不含~~ | `rm -f /data/ShellCrash/.start_error` ✅ | **现在允许** |

## 技术亮点

### 1. 从宽泛到精准的演进

```
第 1 代: /\brm\s+/                           → 拦截所有 rm
       ❌ 无法清理临时文件

第 2 代: /\brm\s+.*\/(data|etc|...)/         → 拦截 /data/xxx
       ❌ 仍然太宽泛，误拦截 /data 下的文件

第 3 代: /\brm\s+...\/data\s|\/data$/        → 只拦截删除 /data 目录本身
       ✅ 精准拦截，允许安全操作 [最终版]
```

### 2. 正则表达式设计原则

```
目标: 区分 "删除目录本身" vs "删除目录内文件"

方案: 使用 \s（空格） | $（行尾） 来标记目录结束
- \s: 表示下一个参数，即删除多个对象（整个目录被删除）
- $: 表示命令结束，即删除这个目录本身
```

## 关键学习

### 黑名单设计的两个原则

1. **精确性**: 不能过度宽泛，否则会误拦截正常操作
2. **清晰性**: 黑名单应明确表达"什么被禁止"，而不是"什么可能危险"
