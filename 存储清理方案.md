## 问题回顾

路由器根文件系统满载 (100%) 导致 ShellCrash 无法启动，网关完全瘫痪。主要原因：

1. Country.mmdb (8.2M) - 地理位置数据库
2. GeoSite.dat (4M) - 域名地理分布数据库  
3. ui 资源 (2.5M) - Web 前端资源
4. 其他日志和缓存

**解决方案已实施**:
- ✅ 删除无关文件释放 13M
- ✅ 实施自动清理机制（每日凌晨 02:00）
- ✅ 添加监控告警（使用率 > 80%）
- ✅ 提供手动清理 API

## 三层防护方案

### 第一层：自动定时清理
**时间**: 每日凌晨 02:00
**清理项目**:
- 7 天前的日志文件
- 过期的缓存数据库 (cache.db)
- 临时文件 (/tmp/*.tmp)

**文件**: `src/services/storageCleanupService.js`

```javascript
// 每日凌晨 02:00 自动执行
StorageCleanupService.startDailyCleanup();

// 日志示例：
// 🧹 开始清理路由器存储空间...
// ✓ 已清理 7 天前的日志
// ✓ 已清理缓存数据库
// ✓ 已清理临时文件
// 📊 当前 /data 使用率: 23%
```

### 第二层：智能告警
**触发条件**: 使用率 > 80%
**告警行为**:
- 记录 WARNING 级日志
- 建议删除大型数据库文件

```
⚠️ 警告: 存储使用率高于 80%，建议删除无用文件
   可选删除：Country.mmdb (8M) 和 GeoSite.dat (4M)
```

### 第三层：手动清理 API
**端点**: `POST /api/cleanup`
**用途**: 紧急清理或诊断时手动触发

```bash
# 示例：从 NAS 上手动触发清理
curl -X POST http://localhost:3000/api/cleanup
```

## 可选优化：删除大型数据库

如果使用率仍然高于 50%，可删除以下非核心文件：

```bash
# 删除地理位置数据库（8.2M）
rm -f /data/ShellCrash/Country.mmdb

# 删除域名地理分布数据库（4M）
rm -f /data/ShellCrash/GeoSite.dat

# 删除前端资源（2.5M）
rm -rf /data/ShellCrash/ui

# 删除这些文件后：
# - Country.mmdb：影响 Clash 的地理位置精准度
# - GeoSite.dat：可能影响某些地理分流规则
# - ui：如果你使用 Clash 的官方 Web UI 会失效（但项目有独立的 Web 端）
```

## 故障排查

**问题**: 清理任务没有执行
**检查**:
1. 容器日志是否有 StorageCleanup 输出
2. 路由器 SSH 连接是否正常
3. 清理权限是否足够

**问题**: 使用率不下降
**解决**:
1. 手动删除 Country.mmdb 和 GeoSite.dat
2. 检查是否有其他大型日志文件
3. 清理 Clash 的历史记录或配置缓存
