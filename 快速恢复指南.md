# 🔧 路由器快速恢复指南（手动步骤）

## 情况
- 路由器：192.168.31.1（小米路由器，aarch64）
- CPU: ARM 64-bit
- 问题：ShellCrash 需要重新安装

## 最快方案：通过 OpenWrt Web UI 安装

### 步骤 1：打开路由器管理界面
```
在浏览器中打开：http://192.168.31.1
用户名：admin
密码：(你的路由器密码)
```

### 步骤 2：找到应用/插件菜单
- 点击左侧菜单 → "应用" 或 "插件"
- 或者搜索 "ShellCrash"、"Clash"、"Mihomo"

### 步骤 3：安装 ShellCrash
- 如果找到，直接点击"安装"
- 等待 3-5 分钟安装完成
- 系统会自动启动 Clash

### 步骤 4：验证安装
在电脑上运行：
```bash
ssh root@192.168.31.1
# 输入密码
pidof CrashCore && echo '✅ 已启动' || echo '❌ 未启动'
```

---

## 备选方案 A：通过命令行快速安装

如果你已经有 SSH 连接权限，直接运行：

```bash
# 1. SSH 连接到路由器
ssh root@192.168.31.1

# 2. 下载安装脚本（路由器上执行）
cd /tmp
curl -k -L -o install.sh 'https://raw.githubusercontent.com/juewuy/ShellCrash/master/install.sh'

# 3. 执行安装
sh install.sh

# 4. 按提示操作（一般选择默认即可）
```

---

## 备选方案 B：手动上传二进制（如果你有备份）

```bash
# 如果你之前有备份的 Clash 二进制文件

# 1. 上传到路由器
scp -o PubkeyAcceptedAlgorithms=+ssh-rsa ~/Downloads/Clash root@192.168.31.1:/data/ShellCrash/

# 2. SSH 连接
ssh -o PubkeyAcceptedAlgorithms=+ssh-rsa root@192.168.31.1

# 3. 设置权限
chmod +x /data/ShellCrash/Clash

# 4. 启动
sh /data/ShellCrash/emergency_start.sh
```

## 快速验证清单

**安装完成后运行这些命令验证**：

```bash
# 1. 检查进程
pidof CrashCore
# 期望输出：一个数字（进程 ID）

# 2. 检查监听端口
netstat -nlt | grep -E '(7890|1053|9999)'
# 期望输出：看到这些端口在监听

# 3. 测试代理连接
curl -I -x http://127.0.0.1:7890 http://cp.cloudflare.com/generate_204
# 期望输出：HTTP/1.1 204 No Content

# 4. 查看日志
tail -20 /data/ShellCrash/run.log
```

## 🎯 建议操作顺序

### 立即尝试（最快）
1. **打开路由器 Web UI** → 应用商店搜索 ShellCrash → 点击安装
2. 等待 5 分钟
3. 在电脑上运行 `pidof` 命令验证

### 如果 Web UI 找不到 ShellCrash
1. 在路由器上执行命令行安装脚本
2. 或上传备份的二进制文件

---

## 🚨 故障排查

| 问题 | 原因 | 解决 |
|------|------|------|
| Web UI 打不开 | 路由器离线 | 检查网线/WiFi 连接 |
| 找不到 ShellCrash | 应用源有限制 | 用命令行安装方案 |
| 安装失败 | 存储满 | 已清理，应该没问题 |
| 启动失败 | 缺少依赖 | 查看日志：`tail /data/ShellCrash/run.log` |
