- ❌ `| sh`, `| bash` - Shell 注入
- ❌ `| nc -l` - 反向 Shell

## 安全架构

### 三层防御

```
输入命令
    ↓
第 1 层：黑名单检查
└─ 拦截已知危险操作（chmod, rm -rf /, 重启等）
    ↓ 通过
第 2 层：白名单检查  
├─ 绝对路径检查 → /tmp/ShellCrash/, /data/ShellCrash/
├─ 命令名检查 → pidof, grep, sed 等
└─ 路径前缀匹配 → /etc/init.d/
    ↓ 通过
第 3 层：实际执行
└─ execFile() - 避免 shell 注入
```

### 关键改进

1. **黑名单精确化**: 从宽泛的模式改为精确针对危险操作
2. **白名单完整化**: 从不完整到覆盖所有实际使用的命令
3. **绝对路径支持**: 添加了路径前缀白名单机制
4. **分层检查**: 优先检查绝对路径，避免过度宽松
