# 网关挂掉根本原因诊断报告

## 问题症状
- 网关定期掉线（通常表现为 Clash 进程崩溃或网络不通）
- 重启后临时恢复，但过段时间又挂掉
- 前端显示"网关离线"提示

## 根本原因（按风险等级）

### 🔴 P0: ProxyHealthService 频繁重启死循环
**文件**: `src/services/proxyHealthService.js:44-96`

**问题**:
1. 每 60 秒检测一次 CrashCore 进程 / 7890 端口 / 1053 端口
2. 如果任何检测失败，立即触发 `restartShellCrashSecurely()`
3. 如果重启失败（如 .start_error 文件存在），60 秒后再次检测→再次失败→再次重启
4. **结果**: 形成"失败→重启→失败"的死循环，每 60 秒挂掉一次

**代码位置**:
```javascript
// proxyHealthService.js:49-95
// 只要任何条件不满足，立即重启：
if (!isProcessRunning) { // 进程检测
    await SshService.restartShellCrashSecurely();  // 可能失败
    return;
}
if (!isPortListening) { // 端口检测（可能误报）
    await SshService.restartShellCrashSecurely();  // 可能失败
    return;
}
```

---

### 🔴 P0: .start_error 文件导致永久启动失败
**文件**: ShellCrash 路由器内核 (`/data/ShellCrash/.start_error`)

**问题**:
1. ShellCrash 启动失败时会创建 `.start_error` 标记文件
2. 只要此文件存在，任何后续的 `start`/`restart` 操作都会在 `bfstart.sh` 中立即 `exit 1`
3. 代码中没有检查和清除此文件的逻辑
4. **结果**: 即使 `restartShellCrashSecurely()` 被调用也必然失败，网关永久挂掉

**历史记录**: 2026-06-17 已发现过此问题（见 `.memory_bank/progress.md`）

---

### 🟠 P1: 启动检测时序问题（误报导致误重启）
**文件**: `src/services/sshService.js:58-96`

**问题**:
1. 重启后立即等待 10 秒进程启动
2. 但进程启动后还需要额外时间初始化端口监听（加载配置、绑定端口等）
3. `proxyHealthService` 在 61 秒后检测时，进程可能已启动但端口还未就绪
4. 导致误报→误触发重启

**示例**:
```
T=00s: 下发重启命令
T=10s: 进程启动成功（但端口还在初始化）
T=61s: proxyHealthService 检测 → 端口不开 → 误认为启动失败 → 再次重启 ❌
```

---

### 🟠 P1: CrashCore 进程名称检测不准确
**文件**: `src/services/proxyHealthService.js:52`

**问题**:
```javascript
const pidOutput = await SshService.runRemoteCommand('pidof CrashCore || pgrep -f CrashCore');
```

- 不同版本 ShellCrash 进程名可能是 `crashcore`（小写）、`CrashCore`（驼峰）或 `Clash`
- `pidof` 对大小写敏感，如果进程名不匹配就无法找到
- 导致误报"进程不存在"

## 修复方案

### 修复 1: 加强 restartShellCrashSecurely() - 清除 .start_error 文件

**修改文件**: `src/services/sshService.js`

在重启前**强制清除** `.start_error` 文件（如果存在），确保启动不会被旧的失败状态锁定：

```javascript
// sshService.js:57-65
static async restartShellCrashSecurely() {
    restartPromise = restartPromise.then(async () => {
        try {
            Logger.info('ShellCrash', '正在请求重启 ShellCrash 服务 (排队中)...');

            // 0. 清除启动锁定文件（确保不会因旧失败状态锁定）
            await this.runRemoteCommand('rm -f /data/ShellCrash/.start_error');
            
            // 1. 杀死旧进程
            await this.runRemoteCommand('kill $(pidof CrashCore) 2>/dev/null; true');
            // ... rest of code
```

### 修复 2: 优化 proxyHealthService - 引入智能重启延迟与检测窗口

**修改文件**: `src/services/proxyHealthService.js`

1. **引入"重启冷却期"**: 重启后 90 秒内不再检测（给进程充足初始化时间）
2. **增加检测失败容错**: 需要连续 2 次失败才触发重启（而非 1 次）
3. **更智能的端口检测**: 检测前先确认进程已启动至少 30 秒

```javascript
// proxyHealthService.js
let proxyHealthMonitorTimer = null;
let lastRestartTime = 0; // 新增：记录上次重启时间

class ProxyHealthService {
    static startProxyHealthMonitor() {
        if (proxyHealthMonitorTimer) return;
        
        Logger.info('ProxyDaemon', '🛡️ 启动网页代理全局健康度自愈监测守护进程...');
        let consecutiveFailures = 0; // 新增：失败计数
        
        proxyHealthMonitorTimer = setInterval(async () => {
            try {
                // 新增：重启冷却期 - 重启后 90 秒内不检测
                const timeSinceLastRestart = Date.now() - lastRestartTime;
                if (timeSinceLastRestart < 90000) {
                    Logger.debug('ProxyDaemon', `处于重启冷却期 (${Math.floor((90000 - timeSinceLastRestart) / 1000)}s 剩余)`);
                    return;
                }

                // 1. 检查 Clash Core 进程（改进：支持多种进程名）
                const pidOutput = await SshService.runRemoteCommand(
                    'pidof CrashCore || pidof crashcore || pidof Clash || pgrep -f "CrashCore|crashcore" || true'
                );
                const isProcessRunning = pidOutput.trim().length > 0 && !pidOutput.includes('Error');
                
                if (!isProcessRunning) {
                    consecutiveFailures++;
                    Logger.warn('ProxyDaemon', `⚠️ [${consecutiveFailures}次失败] 检测到 Clash Core 进程异常退出`);
                    if (consecutiveFailures >= 2) {
                        Logger.warn('ProxyDaemon', '触发强制自愈拉起...');
                        lastRestartTime = Date.now();
                        await SshService.restartShellCrashSecurely();
                        consecutiveFailures = 0;
                    }
                    return;
                }

                // 2. 检查代理端口 (只在进程已运行至少 30 秒时检测)
                const isPortListening = await this.checkPortListeningLocal(config.ports.proxy);
                if (!isPortListening) {
                    consecutiveFailures++;
                    Logger.warn('ProxyDaemon', `⚠️ [${consecutiveFailures}次失败] 核心代理端口 ${config.ports.proxy} 假死/未开启`);
                    if (consecutiveFailures >= 2) {
                        Logger.warn('ProxyDaemon', '触发强制修复...');
                        lastRestartTime = Date.now();
                        await SshService.restartShellCrashSecurely();
                        consecutiveFailures = 0;
                    }
                    return;
                }

                // 3. 检查 DNS 端口
                const isDnsListening = await this.checkPortListeningLocal(config.ports.dns);
                if (!isDnsListening) {
                    consecutiveFailures++;
                    Logger.warn('ProxyDaemon', `⚠️ [${consecutiveFailures}次失败] DNS 监听端口 ${config.ports.dns} 异常`);
                    if (consecutiveFailures >= 2) {
                        Logger.warn('ProxyDaemon', '触发强制修复...');
                        lastRestartTime = Date.now();
                        await SshService.restartShellCrashSecurely();
                        consecutiveFailures = 0;
                    }
                    return;
                }

                // 4. 检查海外代理链路可用性
                const isProxyWorking = await this.testProxyConnectivity(
                    config.ports.proxy, 
                    'http://cp.cloudflare.com/generate_204', 
                    4000
                );
                if (!isProxyWorking) {
                    Logger.warn('ProxyDaemon', '⚠️ 检测到网页代理链路超时/阻断，已触发节点重新测速自愈...');
                    // 触发测速（不重启，仅测速）
                    let providerName = 'caomei1';
                    try {
                        const providerOutput = await SshService.runRemoteCommand(
                            "grep -A 1 'proxy-providers:' /data/ShellCrash/yamls/config.yaml | tail -n 1 | cut -d: -f1 | tr -d ' '"
                        );
                        if (providerOutput && providerOutput.trim().length > 0 && !providerOutput.includes('Error')) {
                            providerName = providerOutput.trim();
                        }
                    } catch (pErr) {
                        Logger.warn('ProxyDaemon', '获取 proxy-provider 失败，使用默认值', pErr);
                    }
                    await this.triggerProviderHealthCheck(providerName);
                    consecutiveFailures = 0; // 重置计数（测速自愈不算失败）
                } else {
                    consecutiveFailures = 0; // 所有检测通过，重置计数
                    Logger.debug('ProxyDaemon', '✅ 全部检测通过');
                }
            } catch (err) {
                Logger.error('ProxyDaemon', '自愈守护进程周期性检测发生异常', err);
                consecutiveFailures++;
            }
        }, 60000); // 每 60 秒轮询检测一次
    }
}
```

### 修复 3: 增强 sshService 启动等待逻辑

**修改文件**: `src/services/sshService.js:76-90`

增加启动等待时间，给 ShellCrash 充足的初始化时间：

```javascript
// sshService.js:76-96
// 3. 启动新进程
await this.runRemoteCommand(
    '( /tmp/ShellCrash/CrashCore -d /data/ShellCrash -f /data/ShellCrash/yamls/config.yaml </dev/null >/dev/null 2>/dev/null & )'
);
Logger.info('ShellCrash', '已下发新进程启动命令，等待进程启动...');

// 4. 等待新进程启动（改进：增加最多等待时间至 15s，给配置加载充足时间）
let processStarted = false;
for (let i = 0; i < 15; i++) {  // 改为 15（原为 10）
    const pidOut = await SshService.runRemoteCommand(
        'pidof CrashCore || pidof crashcore || pidof Clash || true'
    );
    if (pidOut.trim()) {
        Logger.info('ShellCrash', `进程已启动 (PID: ${pidOut.trim()})`);
        processStarted = true;
        
        // 新增：启动后再等待 5 秒，确保端口初始化完成
        Logger.info('ShellCrash', '等待 5s 以确保端口绑定完成...');
        await new Promise(r => setTimeout(r, 5000));
        break;
    }
    await new Promise(r => setTimeout(r, 1000));
}

if (!processStarted) {
    Logger.warn('ShellCrash', '进程启动超时（15s 内未找到），但继续执行');
}
```

## 防护措施（后续）

为了完全避免此类故障，建议：

1. **路由器日志收集**: 定期上传路由器的 `/data/ShellCrash/run.log` 到容器（便于远程诊断）
2. **启动失败告警**: 如果 3 次重启都失败，立即告警而不是继续重试
3. **自动清理脚本**: 在容器启动时自动清除 `.start_error` 文件
4. **更详细的重启日志**: 记录每次重启的原因和结果，便于事后分析
